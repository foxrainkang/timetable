# Google Apps Script로 시간표 배포하기

## 개요
이 가이드는 시간표를 Google Apps Script를 통해 웹 앱으로 배포하는 방법을 설명합니다.

## 준비물
- Google 계정
- 웹 브라우저

## 배포 단계

### 1단계: Google Apps Script 프로젝트 생성

1. [Google Apps Script](https://script.google.com)에 접속
2. **새 프로젝트** 클릭
3. 프로젝트 이름을 "시간표" 또는 원하는 이름으로 변경

### 2단계: 코드 작성

**GoogleAppsScript.gs** 파일의 전체 코드를 복사하여:

1. Apps Script 편집기의 `Code.gs`에 모두 삭제 후 붙여넣기
2. **저장** (Ctrl+S)

**이 코드의 특징:**
- `doGet()` 함수: 웹 앱 진입점
- `getHtmlContent()` 함수: HTML 전체 코드 반환
- 모든 CSS와 JavaScript가 한 파일에 포함됨
- sessionStorage 사용 (새로고침 시까지 데이터 유지)

### 3단계: 배포

1. **배포** 버튼 클릭 (우측 상단)
2. **새 배포** 선택
3. 설정:
   - **유형**: 웹 앱
   - **실행**: 나 (또는 본인 계정)
   - **다음 사용자로 액세스**: 모든 사용자
4. **배포** 클릭

### 4단계: 웹 앱 URL 획득

배포 후 나타나는 URL을 복사하여 브라우저에서 접속!

예: `https://script.google.com/macros/d/{PROJECT_ID}/userweb`

## 데이터 저장 방식

**주의**: Google Apps Script 웹 앱은 sessionStorage를 사용하므로:
- ✅ 브라우저의 현재 세션에서는 데이터 유지
- ❌ 새로운 탭/브라우저에서는 데이터 초기화
- 💾 영구 저장이 필요하면 JSON 백업 기능 사용

### 데이터 영구 저장 버전 (선택사항)
Google Sheets와 연동하여 영구 저장하는 버전도 가능합니다.
필요하시면 별도로 요청해주세요!

## 문제 해결

### 배포 후 빈 화면이 보일 때
→ 브라우저 새로고침 (F5 또는 Ctrl+R)

### 강의 추가가 안 될 때
→ 브라우저 개발자 도구 (F12) → Console 탭에서 오류 확인

### 새 탭에서 데이터가 없을 때
→ 정상입니다. JSON 백업으로 복원하거나, 새 탭에서 다시 입력하세요

## 추가 기능 추가

### 커스터마이징 방법
1. Apps Script 편집기에서 코드 수정
2. **배포** → **배포 관리**
3. 기존 배포 수정 또는 새 버전으로 배포
